第一步：初始化项目
bash# 1. 创建项目
npm create vite@latest memory-palace -- --template react
cd memory-palace

# 2. 安装依赖（极简，不需要额外库）
npm install

# 3. 清理默认文件
rm src/App.css src/assets/react.svg

第二步：本地运行
bashnpm run dev
# 浏览器打开 http://localhost:5173
热更新是即时的，改一个组件文件保存就刷新，不影响其他层。

第三步：构建产物
bashnpm run build
# 生成 dist/ 目录，里面是可直接部署的静态文件

第四步：上传 GitHub
bash# 在项目根目录
git init
git add .
git commit -m "feat: init memory palace"

# 去 GitHub 新建仓库（不要勾选 README），然后：
git remote add origin https://github.com/你的用户名/memory-palace.git
git branch -M main
git push -u origin main

第五步：一键部署生成链接（推荐 Vercel）
方案A — Vercel（最快，30秒）：

去 vercel.com 用 GitHub 登录
点 "Add New Project" → 选你的仓库
Framework 自动识别为 Vite，直接点 Deploy
得到 https://memory-palace-xxx.vercel.app 链接
以后每次 git push 自动重新部署

方案B — GitHub Pages（免费但需配置）：
在 vite.config.js 加一行：
jsexport default {
  base: '/memory-palace/', // 仓库名
}
然后：
bashnpm install -D gh-pages
# package.json 的 scripts 里加：
# "deploy": "gh-pages -d dist"
npm run build && npm run deploy
链接是 https://你的用户名.github.io/memory-palace/

迁移顺序建议
建议按这个顺序拆，每完成一步都能跑通：
1. 搭骨架         App.jsx + Ocean.jsx，能渲染4个空层
2. 迁移样式        把 CSS 按模块拆好，全部 import 进来
3. 迁移数据        data/ 下的几个 js 文件，纯搬运
4. LayerIntro     最简单，纯静态 HTML
5. LayerSticky    便利贴 + 拖拽（复用 v12 已修好的版本）
6. LayerShelf     书架 + 书卡 hover（JS 控制版）
7. LayerLibrary   Canvas 图谱（用 useEffect + useRef）
8. SummonOverlay  最后，独立浮层
9. DepthGauge     右侧导航 + 过场动画
